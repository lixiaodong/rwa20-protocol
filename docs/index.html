Why RWA20

The $500 Trillion Problem

Global real-world assets are worth ~$500 trillion. Less than 0.01% is tokenized. The bottleneck is not technology — it is compliance.

❌ Today's Fragmented Reality
  • Every issuer rebuilds compliance from scratch
  • Each jurisdiction requires a separate legal wrapper
  • On-chain and off-chain legal proofs cannot be linked
  • Token transfers bypass compliance checks
  • No standard interoperability between RWA platforms
  • Investors cannot transfer RWA tokens cross-border
✅ RWA20 Changes Everything
  • Compliance is built into the token's transfer hook — unforgeable
  • Plug-in jurisdiction modules: SG · CH · AE · LI · MY · US
  • On-chain legal attestations via EAS + Kleros + native registries
  • One protocol works across all asset classes
  • Open standard: add a jurisdiction with one contract
  • Zero-fee bootstrap phase — maximize adoption first
Core Protocol

Built for the Long Game

Six layers of protocol infrastructure, designed to become an industry standard — not just another DeFi product.

🔒

Compliance-Native Transfer

Legal checks live inside ERC20's _update() hook. Every transfer — DEX, wallet, bridge — is compliance-checked with zero bypass risk.

🌍

Multi-Jurisdiction Engine

OR-logic: investors qualify if they meet any enabled jurisdiction. Adding a new country requires deploying exactly one contract.

⚖️

On-Chain Legal Attestations

Property titles, court orders, KYC records, regulatory approvals — attested on-chain via EAS schemas and Kleros arbitration.

💰

Snapshot Revenue Distribution

O(1) per-holder claims using ERC20Votes snapshots. No iteration, no gas explosion. Rent, yield, dividends — all automated.

📈

Bonding Curve Launchpad

Fair-launch price discovery for any asset. Linear bonding curve with configurable base price and slope. No VCs, no presale.

🔄

DEX Wrapper (wRWA20)

Compliant tokens become freely tradeable via a wrapped version. The wrapper holds compliance; the inner token stays pristine.

🗳️

Governance Ready

Built on ERC20Votes + AccessControl. Asset holders vote on distributions, compliance parameters, and protocol upgrades on day one.

⚡

Protocol Fee Manager

Hard-capped fees (issuance ≤1%, distribution ≤2%, swap ≤0.3%). All start at 0% in bootstrap phase. Fee routing: 70% treasury / 20% ecosystem / 10% stakers.

🔮

Oracle Price Feeds

Pluggable oracle interface with Chainlink adapter. Asset NAV updated on-chain; bonding curve price tracks real-world valuations.

FAQ

Common Questions

Answers optimized for developers, lawyers, and asset managers exploring RWA tokenization.

RWA20 is an open-source smart contract protocol and emerging standard for tokenizing real-world assets (RWA) on Ethereum. Unlike closed platforms (Centrifuge, Maple, Ondo), RWA20 is a permissionless standard — similar to how ERC20 standardized fungible tokens. Any developer or asset manager can deploy an RWA20-compliant token with native legal compliance baked into the token transfer mechanism itself, across multiple jurisdictions simultaneously.

The compliance check is embedded inside ERC20's internal _update() function — the lowest-level transfer hook. Every transfer calls MultiJurisdictionCompliance, which uses OR-logic: a transfer is approved if both sender and receiver each qualify under at least one enabled jurisdiction. Jurisdiction modules are independent contracts (SGJurisdiction, USJurisdiction, etc.) that enforce their local securities law. Adding a new country requires deploying one contract and calling registerJurisdiction("XX", moduleAddress).

On-chain legal attestations are cryptographically signed proofs — stored on the Ethereum blockchain — that link a digital token to its real-world legal basis. RWA20 supports 9 attestation types: property titles, corporate records, court orders, audit certificates, KYC verification, regulatory approvals, valuation reports, custody proofs, and legal opinions. These are bridged from the Ethereum Attestation Service (EAS), Kleros decentralized arbitration, and jurisdiction-specific bodies (ADGM Courts, ACRA, FMA). This creates a traceable legal chain from token to real-world asset.

Yes. The protocol is MIT licensed and completely free during the bootstrap phase (Phase 0). Protocol fees are hard-capped in code: issuance ≤1%, revenue distribution ≤2%, DEX swap ≤0.3% — and all start at 0%. The business model follows the open-source playbook: build credibility as a standard first, then introduce optional fee tiers for institutional users in Phase 2-3. The goal is to become the ERC20 of RWA — a standard so widely adopted that commercial services naturally grow around it.

The protocol is designed for legal experts to contribute without writing Solidity. Each jurisdiction module is a standalone contract with a clear interface. Legal researchers can: (1) review and improve existing jurisdiction logic, (2) contribute new jurisdiction modules by filing a GitHub issue describing the regulatory framework, (3) become a trusted attestor within the LegalAttestationRegistry, (4) co-author the jurisdiction-specific compliance documentation. See CONTRIBUTING.md for the full 5-step process.

RWA20 is asset-class agnostic. Any real-world asset that can be legally represented as a fungible token is supported: commercial real estate (rental yield), agricultural commodities (Durian farms, timber), private equity funds, trade finance receivables, government bonds, infrastructure projects, IP royalties, carbon credits, and more. The bonding curve launchpad is particularly suited for assets with progressive price discovery, while the revenue distributor handles any recurring yield stream.
